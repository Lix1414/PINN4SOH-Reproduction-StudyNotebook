# Cycle Index归一化对PINN4SOH训练结果的影响解读报告

## 1. 报告目的

本报告比较PINN4SOH在XJTU六种工况上的两组完整实验：

- 旧版：16项统计特征归一化，`cycle index`保持原始循环编号；
- 新版：16项统计特征与`cycle index`全部进行逐电池Min-Max归一化；
- 两版均运行6种工况，每种工况10次，共60次；
- 核心模型、数据文件、训练/测试电池、损失函数和训练超参数保持一致。

比较结果支持以下判断：遗漏 `cycle index` 归一化是此前复现结果与作者报告值产生明显差距的主要影响因素之一。由于两组实验未使用配对随机种子，本报告将其视为强一致性证据，而不是严格的单因素因果证明。

## 2. 实验条件

| 项目 | 设置 |
|---|---|
| 数据集 | 作者仓库提供的XJTU预提取CSV |
| 工况 | 2C、3C、R2.5、R3、RW、satellite |
| 重复次数 | 每种工况10次 |
| 随机种子 | 未固定 |
| 最大训练轮数 | 200 epoch |
| Early Stop | 验证MSE连续21轮未改善后停止 |
| Warmup | 30 epoch |
| Batch size | 256 |
| 学习率 | warmup 0.002、base 0.01、final 0.0002 |
| 动力学网络学习率 | 0.001 |
| 损失权重 | `alpha=0.7`、`beta=0.2` |

两版输入数据数量完全一致，差异只集中在`cycle index`是否参与归一化。

## 3. MAPE与RMSE对比

| 工况 | 旧版MAPE | 新版MAPE | MAPE降低 | 旧版RMSE | 新版RMSE | RMSE降低 |
|---|---:|---:|---:|---:|---:|---:|
| 2C | 1.3330% | 0.7029% | 47.3% | 0.014792 | 0.009068 | 38.7% |
| 3C | 1.5485% | 1.2039% | 22.3% | 0.017310 | 0.013077 | 24.5% |
| R2.5 | 1.2258% | 0.8510% | 30.6% | 0.015661 | 0.010232 | 34.7% |
| R3 | 0.9148% | 0.7652% | 16.4% | 0.013776 | 0.011196 | 18.7% |
| RW | 1.7610% | 1.1117% | 36.9% | 0.023437 | 0.013194 | 43.7% |
| satellite | 0.9889% | 0.6402% | 35.3% | 0.013904 | 0.010856 | 21.9% |

六种工况的 MAPE 和 RMSE 均下降，表明改善具有跨工况一致性，不像是由某一次随机初始化单独造成；但若要严格量化因果贡献，仍建议补充配对随机种子的消融实验。

主要观察包括：

1. 2C的MAPE下降47.3%，是相对改善最大的工况；
2. RW的RMSE下降43.7%，说明原始尺度的循环编号对该工况影响尤其明显；
3. R3原本已经具有较低误差，但归一化后仍继续改善；
4. 没有任何工况因归一化`cycle index`而退化。

## 4. 随机训练稳定性对比

| 工况 | 旧版MAPE标准差 | 新版MAPE标准差 | 标准差变化 |
|---|---:|---:|---:|
| 2C | 0.2024% | 0.1036% | 降低48.8% |
| 3C | 0.1477% | 0.1067% | 降低27.8% |
| R2.5 | 0.2031% | 0.1096% | 降低46.0% |
| R3 | 0.1292% | 0.0883% | 降低31.7% |
| RW | 0.4294% | 0.1686% | 降低60.7% |
| satellite | 0.1002% | 0.0498% | 降低50.3% |

所有工况的标准差均下降，这与“归一化改善平均精度并降低随机训练波动”的解释一致。由于两组实验未共享配对种子，不能仅凭该表把全部稳定性改善归因于单一预处理变化。

RW 的标准差下降最明显，从 0.4294% 降至 0.1686%。该现象提示输入尺度失衡可能是旧版 RW 波动较大的重要因素，但不排除随机初始化和训练过程差异的共同影响。

## 5. 为什么Cycle Index会产生如此大的影响

### 5.1 旧版输入尺度失衡

模型输入由以下部分组成：

```text
16项电池统计特征 + cycle index
```

旧版中，16项统计特征已经被归一化到约`[-1,1]`，但`cycle index`仍可能取值：

```text
0, 1, 2, ..., 400，甚至更大
```

因此，循环编号的数值尺度可能比其他输入大数百倍。进入神经网络后，它会：

- 在第一层线性组合中占据主导；
- 压低其他电压、电流、容量和时间特征的相对贡献；
- 造成不同输入方向的梯度尺度不平衡；
- 增加优化器寻找稳定参数区域的难度；
- 放大不同随机初始化之间的训练差异。

### 5.2 新版恢复输入尺度平衡

逐电池Min-Max归一化后，循环编号被映射到`[-1,1]`附近：

```text
寿命前期 → 接近-1
寿命中期 → 接近0
寿命后期 → 接近1
```

此时，模型可以同时使用循环进度与16项电池状态特征，不再由一个数值远大的输入控制主要激活和梯度。

从表示学习角度看，归一化后的`cycle index`不再只是绝对循环编号，而更接近“该电池当前处于自身完整退化轨迹的相对位置”。这与SOH变化高度相关，因此能显著提高预测精度。

## 7. 原始源码依据

原始数据加载器先插入循环编号：

```python
df.insert(df.shape[1] - 1, 'cycle index', np.arange(df.shape[0]))
```

随后选择除最后一列`capacity`以外的全部列：

```python
f_df = df.iloc[:, :-1]
```

并执行Min-Max归一化：

```python
f_df = 2 * (f_df - f_df.min()) / (f_df.max() - f_df.min()) - 1
df.iloc[:, :-1] = f_df
```

因为`cycle index`位于`capacity`之前，所以它包含在`df.iloc[:, :-1]`中。新版clean代码恢复了这一行为；旧版将其排除，属于对原始数据流的偏离。

## 8. 方法学风险：可能使用未来寿命信息

虽然逐电池归一化`cycle index`忠于公开源码，但需要单独披露一个重要风险。

对一节完整测试电池执行Min-Max归一化，需要知道该电池循环编号的最大值。归一化后的循环位置实际上是：

```text
当前循环 / 该电池完整生命周期长度
```

在真实在线预测中，模型通常不知道一节正在运行的电池最终会有多少个循环。因此，这种预处理方式可能间接使用测试电池的未来寿命长度，使任务比真实在线SOH预测更容易。

应当区分两个目标：

| 目标 | 建议处理 |
|---|---|
| 忠实复现公开源码 | 保留逐电池`cycle index`归一化 |
| 构建真实在线模型 | 使用训练集固定尺度或已知设计上限，禁止使用测试电池最终循环数 |

对于后续创新模型，建议同时报告：

1. 论文复现协议：逐电池归一化；
2. 无未来信息协议：使用训练集固定循环尺度；
3. 两种协议之间的性能差异。

这样既能与原论文进行可比对照，也能进一步评估模型在接近真实部署条件时的可信度；仅完成该对照并不能单独证明部署有效性。

## 9. 最终结论

本次受控对比表明：

> 旧版遗漏 `cycle index` 归一化，使循环编号的数值尺度远大于其他 16 项输入。受控对比结果表明，这种尺度失衡与更高的预测误差和随机波动相关。

恢复原始源码的归一化行为后：

- 六种工况的MAPE降低16.4%至47.3%；
- 六种工况的RMSE降低18.7%至43.7%；
- 所有工况的随机训练标准差下降；
- 结果整体与作者提供的实验结果处于接近量级。

因此，包含 `cycle index` 归一化的新结果应作为当前正式复现结果。未归一化版本保留为历史基线，用于展示输入尺度处理与 PINN4SOH 训练稳定性、预测精度之间的关系。

同时，正式报告必须注明逐电池循环归一化可能使用完整寿命长度这一方法学限制，避免将论文复现性能直接等同于真实在线预测性能。

两组实验的机器可读汇总分别见 [`cycle_normalized_summary.json`](results/cycle_normalized_summary.json) 和 [`cycle_unscaled_summary.json`](results/cycle_unscaled_summary.json)。完整单次模型和预测数组因体积较大未纳入 Git。
